Install Docker and Create Images

Written by Pepper Targaryen with ♥ on November 9, 2019 in 教程

Docker is a software platform that allows you to build, test, and deploy applications quickly. In this post, we are going to install Docker and create an image for the test environment.

Install Docker

Docker supports both Linux and Windows.

For Linux:

This link should give you the tutorial for the installation of the community edition of Docker

For Windows

This link should give you the tutorial for the installation of Docker Desktop for Windows

Hello World

After the installation of Docker for our system, we should be able to test this.

Ubuntu Terminal

sudo docker run hello-world

Windows Powershell

docker run hello-world

Create an Image

First we need to figure out what test environment we require. Here I want to test a Java-based Project so I am going to test this on an Ubuntu and install Java 8. Let’s assume I have a laptop which installed Windows 10.

Virtual Network

This creates a virtual network named hadoop for DNS.

docker network create --driver=bridge hadoop

We can check the network.

docker network ls

Pull Ubuntu

First we search Ubuntu. If the result is not ideal, we should consider changing the image source of Docker.

docker search ubuntu

Download the image with a specified version.

docker pull ubuntu:16.04

Check the images.

docker images

Run a container based on the image.

docker run -it ubuntu:16.04 /bin/bash

Type exit to quit the container

List all the running containers.

docker ps

If a container is closed, we can restart it.

docker start containerId

or

docker start containerName

Then we can enter the container.

docker exec -it containerId /bin/bash

Set up the Environment

Enter the container we started before.

Change the source.

cp /etc/apt/sources.list /etc/apt/sources_init.list
rm /etc/apt/sources.list
echo "This is the source" > /etc/apt/sources.list

And update the changes.

apt update

Now we can install what we want.

apt install openjdk-8-jdk

We can test the Java version.

java -version

Other tools to be installed.

apt install vim
apt install net-tools
apt install nmap

Install SSH

apt-get install openssh-server
apt-get install openssh-client

To make the connection simple, we don’t want to enter password when we try to connect.

cd ~
ssh-keygen -t rsa -P ""

Add the key to authorized_keys.

cat .ssh/id_rsa.pub >> .ssh/authorized_keys

Start SSH service.

service ssh start

We can try to SSH itself.

ssh 127.0.0.1

Add the auto-start command.

vim ~/.bashrc

Insert service ssh start at the end of the file.

Save the container as our own image.

docker commit -m "hadoop" -a "hadoop" containerId ourImageName

Test

We can now create several containers to launch the test.

docker run -it --network hadoop -h "h01" --name "h01" -p 9870:9870 -p 8088:8088 ourImageName /bin/bash

Another one

docker run -it --network hadoop -h "h02" --name "h02" ourImageName /bin/bash

We can transfer our files from Windows laptop to containers.

docker cp fileOrFolderPath containerId:Path

Last Thing

This image I created is available on the Docker Hub, named "peppertargaryen/java8".
